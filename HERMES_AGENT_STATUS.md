# Hermes Agent Status

Current Version: v0.9.0

## Status

Hermes Live Agent Integration Plan

## Purpose

Hermes is the planned live-agent profile layer for THE GRID. It defines specialist roles, autonomy boundaries, approval gates, and future profile placeholders before any external account is connected.

This is preparation only. No Telegram bots, marketplace accounts, payment systems, wallets, trading systems, or customer channels are connected.

## Profiles

- ZENITH: Chief Systems Architect
- REVENUE ARCHITECT: Income systems strategist
- DESIGN FORGE: Original asset creation specialist
- COPY ROOM: Offer copy and customer communication drafting specialist
- SENTINEL: Risk and safety officer

## Autonomy Levels

- Level 1 Draft: prepare drafts, briefs, checklists, and local notes for review.
- Level 2 Research: collect and organize evidence from approved sources.
- Level 3 Build: create local UI, docs, assets, and workflow scaffolds after approval.
- Level 4 Publish requires approval: prepare publishable packages but stop at the human approval gate.
- Level 5 Money/actions forbidden for now: spending, trading, account actions, and external execution are not authorized.

## Required Profile Placeholders

Each Hermes profile includes placeholders for:

- Telegram bot connection
- SOUL.md profile memory
- Playbooks
- Recurring jobs
- Approval gates

These placeholders are intentionally inactive.

## Prohibited

- Autonomous publishing
- Customer or supplier messaging
- Spending money
- Trading or wallet actions
- Account automation
- Credential handling
- ToS-violating scraping
- Fabricated revenue, users, reviews, trades, analytics, or agent activity

## Doctrine

- Prepare identity before integration.
- Define boundaries before autonomy.
- Draft and research before publishing.
- Human approval before external action.
- Mission Commander decides.
