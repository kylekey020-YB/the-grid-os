# Research Router

Version: v3.2.0
Status: Assignment layer / no execution
Date: July 3, 2026

Doctrine source: [[GRID_SYSTEM_PROMPT]]

## Purpose

Research missions are assigned to the best available intelligence.

Research Router never executes. It only assigns.

## Available Platforms

- ChatGPT / ZENITH
- Claude
- Hermes
- Claude Code
- Codex
- Future AI

## Platform Roles

| Platform | Best Use |
| --- | --- |
| ChatGPT / ZENITH | Architecture, strategy, doctrine, prompts, product direction |
| Claude | Deep research, synthesis, analysis, structured critique |
| Hermes | Memory, executive briefs, Mission Records, daily operations |
| Claude Code | Chief Engineer, implementation architecture, deep engineering review |
| Codex | Frontend implementation, repo work, documentation, refactoring, validation |
| Future AI | Specialized research or redundancy after approval |

## Lifecycle

1. Created
2. Assigned
3. Researching
4. Completed
5. Reviewed
6. Archived
7. Rejected

## Assignment Rules

- Assign to the first available platform that best fits the mission.
- If Hermes is unavailable, use the Founding Council fallback chain.
- Codex handles engineering and documentation implementation only within approved scope.
- Claude Code is Chief Engineer and makes engineering decisions from the root GRID repository.
- Research outputs must become Obsidian-compatible Markdown.

## Non-Execution Boundary

Research Router does not:

- Publish.
- Trade.
- Message customers.
- Spend money.
- Connect accounts.
- Connect brokers.
- Trigger APIs.
- Run background jobs.
- Merge code.
- Approve experiments.

Mission Commander approval is required before any irreversible action.
