# THE GRID Design System

Current Version: v0.3.0

This folder defines the visual foundation for THE GRID. It exists so future pages, workstations, and review surfaces feel consistent without copying styles from older screens by guesswork.

The design system is a reference layer, not a feature layer. It does not add business logic, backend behavior, live data, AI automation, or autonomous agents.

## Principles

- Dark interface first.
- High contrast, low clutter.
- Operational surfaces before decorative surfaces.
- Status must be visible at a glance.
- Effects should clarify hierarchy, not distract from evidence.
- New UI should reuse established tokens before inventing new ones.

## Usage

Before building a new page, choose one primary accent family, one status model, one card density, one animation timing tier, and the relevant officer or workstation icon treatment.

Avoid creating one-off palettes, one-off glow strengths, or new card shapes unless the existing system cannot support the workflow.
