# Officer Iconography

Current Version: v0.3.0

Officer identity should be consistent across Mission Control, handoff docs, and future workstation views. Icons identify role and responsibility; they should not imply autonomous authority.

## Officer Roles

| Officer | Role | Icon Direction | Accent |
| --- | --- | --- | --- |
| ZENITH | Systems Architect | Brain, compass, architecture signal | Purple |
| SENTINEL | Safety Monitor | Shield, warning, lock | Red |
| APEX WARDEN | Trading Research Monitor | Activity, chart, signal | Cyan |
| CLU STRIKER | Paper Shot Monitor | Target, activity, timer | Blue |
| GAUNTLET | Backtest Monitor | Database, gauge, test rig | Emerald |
| MERCHANT FORGE | Commerce Validation Monitor | Package, receipt, scale | Gold |
| QUARTERMASTER | Resource Constraint Monitor | Clipboard, archive, inventory | Amber |
| ARCHIVIST | Knowledge Monitor | Book, library, file text | Blue |
| GRID CORE | UI Foundation | Grid, command, circuit | Cyan |

## Rules

- Prefer Lucide icons when a matching symbol exists.
- Icons should support text, not replace critical labels.
- Do not use officer icons to suggest that a role can execute actions.
- Keep icon containers square or circular with stable dimensions.
- Use tooltips for icon-only controls when they become interactive.
