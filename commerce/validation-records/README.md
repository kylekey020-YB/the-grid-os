# Commerce Validation Records

This folder is the Commerce workstation's institutional memory.

Each file here is a validated research outcome — not a business, not a live
opportunity, not a to-do item. These are the results of running a real
opportunity through the full Commerce Validation Pipeline. The work is done.
The conclusion is recorded. It will not be re-researched from scratch.

## Why this exists

Manual research takes 2–4 hours per opportunity. Without records, that work
evaporates and the same ground gets covered again in a future session. These
files make sure that never happens. A "PARKED" record is not a failure — it
is finished work that prevents duplicate effort and keeps the decision log
honest.

## Status definitions

- **GO** — validated and actively being built or sourced
- **PARKED** — validated, documented, not pursued now; conditions for revisit recorded
- **KILLED** — validated as not viable; do not revisit

## How to add a new record

1. Run the opportunity through the full pipeline in [PIPELINE.md](PIPELINE.md)
2. Copy the record template from any existing record
3. Fill every gate — do not skip gates, do not leave "TBD"
4. Set a clear Decision and Outcome
5. Add a row to the table above and commit
